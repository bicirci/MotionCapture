Tool/software: Code Composer Studio

Hello!
I’m very confused about how compile “sensortag_cc2650stk_app” in my SensorTag CC2650. I followed a lot of tutorials and threads of this forum but I can’t do it.
I have a Debug Dev Pack XDS110, Code Composer Studio 7, SmartRF Flash Programmer 2 and BLE-STACK-2-2-1
If I program de precompiled hex file “cc2650stk_sensortag_rel.hex” (C:\ti\simplelink\ble_sdk_2_02_01_18\examples\hex) in my SensorTag, it works well. The green led flashes and my android app detected it.
But if I try to generate the super hex file, the sensortag don’t work.
I follow the steps of user guide:
1. Import the SensorTag project in CCS (C:\ti\simplelink\ble_sdk_2_02_01_18\examples\cc2650stk\sensortag\ccs)
2. Import BIM project in CCS (C:\ti\simplelink\ble_sdk_2_02_01_18\examples\util\bim_extflash\cc2640\ccs)
3. Build the BIM_extflash project [FlashOnly_ST Configuration]
4. Build the Stack project [FlashROM Configuration]
5. Build the Application project [FlashOnlyOAD Configuration]
6. Generate de super hex file:
C:\Python27\Scripts>python hexmerge.py -o sensortag_super.hex -r 0000:1FFFF sensortag_cc2650stk_app.hex:0000:1EFFF sensortag_cc2650stk_stack.hex bim_extflash.hex:0000:1FFFF --overlap=error
I have tried with the values of this thread:
7. Program de sensortag with Smart Flash 2. The process finishes well but the sensor does not work.
I have tried with “Erase All unprotected pages” and “Erase Pages in image”.

I have tried with different advices of this community but nothing work. I am very desperate. Somebody could help me? Thank you very much.
